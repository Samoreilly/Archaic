Two seperate processes|-> Daemon - one that loads trie from disk and stays running
                      |-> Terminal-bound process - process within fish (shell) that sends data to daemon that handles trie
                


Terminal bound process - passes string and current directory to daemon and receives auto completed string


Terminal process sends queries to a daemon over IPC
Daemon performs a trie lookup on the query
If no match, spins up a thread to insert the string into the trie
A second thread runs a weight-based scanner to find autocomplete results
When a new query arrives, the weight-based scanner thread is cancelled
Trie insertion thread is left to finish in the background



- Hashmap storing folder names -> trie using buckets
